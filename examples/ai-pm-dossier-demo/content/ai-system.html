端到端 AI 链路

工单事件
  → 租户与权限上下文
  → 意图 / 风险路由
  → 混合检索 → 重排 → 证据包
  → 提示词编排 → 生成模型 → 结构化校验
  → 引用 / 策略 / PII 检查
  → 人工确认或安全回退
  → 反馈、观测、离线评估

组件与责任

组件责任关键控制失败回退
路由器识别场景与风险版本化规则 + 小模型分类未知场景进入人工模式
RAG构建可引用证据包租户 ACL、时间、版本过滤证据不足则拒答
生成器生成回复草稿系统提示、结构化输出、温度约束一次重试后降级
工具代理只读查询外部系统白名单、参数校验、超时、审计返回人工操作指引
守卫层检查引用、策略与敏感内容规则 + 模型双层检查阻断发送并说明原因

Prompt 管理

  • 系统策略、场景模板、租户语气和输出 schema 分离版本管理。
  • 每次线上响应记录 prompt 版本、模型版本、检索配置和策略版本，禁止记录真实密钥。
  • 提示词变更必须经过回归评测；不能把“措辞优化”视为低风险文案改动。

Agent 边界

E3 · Verified 当前 agent 只允许规划并调用只读工具；任何改变客户权益或账户状态的动作都需要确定性权限校验和人工确认。最大工具步数、时间预算和重复调用检测用于防止循环。

模型策略

按任务路由模型：分类与抽取优先低成本模型，复杂综合使用高能力模型；供应商切换必须通过同一评测集和输出契约，避免“能调用”被误解为“可替换”。
