#系统边界

[客服工作台]
     │ 工单上下文 / 用户操作
     ▼
[Takeover API / Auth / Tenant Policy]
     │
     ├─ [AI Orchestrator] ─ [Model Gateway]
     │        ├─ [Retrieval Service] ─ [Vector + Keyword Index]
     │        ├─ [Prompt Registry]
     │        └─ [Read-only Tool Gateway] ─ [CRM / Order / Status]
     │
     └─ [Events / Traces / Feedback] ─ [Warehouse + Evaluation]

#关键接口契约

边界输入 / 输出所有者保护措施
工作台 → API租户、工单、语言、授权上下文应用工程身份校验、字段最小化、幂等键
编排 → 检索查询、过滤器 → 证据包搜索 / MLACL、版本、超时、可追溯 ID
编排 → 模型版本化消息 → 结构化建议AI 平台schema、预算、重试、供应商策略
编排 → 工具白名单工具与参数集成平台最小权限、审计、熔断、禁止任意 URL

#租户与区域隔离

E3 · 已验证 租户 ID 在 API、检索过滤、缓存键和审计事件中贯穿；区域策略决定数据与模型路由。任何缺少租户上下文的请求都应默认拒绝。

#关键架构决定

  • 模型通过网关接入，业务代码不直接依赖单一供应商 SDK。
  • 提示词与评估版本作为发布制品，支持回放和回滚。
  • 高风险工具不交给自由规划 agent；确定性流程负责授权与确认。
  • 埋点记录引用和版本 ID，不写入密钥或不必要的原始敏感文本。

#架构风险

模型网关是集中故障点；检索 ACL 是跨租户风险点；外部 CRM 限流可能造成级联延迟。三者都需要明确 SLO、熔断和降级演练。
