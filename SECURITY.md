# Security policy

## 报告漏洞

不要在公开 Issue 中披露漏洞、真实令牌或敏感仓库。优先使用 GitHub Private vulnerability reporting，或私下联系仓库所有者，并提供不含真实秘密的最小复现、影响、攻击前提和建议修复。

## 风险模型

- 把令牌、私钥、连接串或个人数据写进手册与搜索索引。
- 把不可信 HTML 注入离线站点。
- 通过资产或链接越出站点目录。
- 远程脚本、样式或媒体引入供应链风险。
- 自动化代理执行危险项目命令。

验证器提供启发式防护，不是完整秘密扫描器、HTML 沙箱或恶意仓库隔离环境。

## 安全使用

- 默认只读审计目标仓库。
- 不运行迁移、部署、删除、上传或生产连接命令，除非用户明确要求并确认范围。
- 不把不可信第三方 HTML 放入 `content/`。
- 公开前人工检查 `content/`、配置、生成页面与搜索索引。
- Mermaid 运行时从可信发行源取得并核对许可证。
