# 项目说明书

## 项目定位

AI 产品经理项目接手 Skill 用于把陌生 AI 项目的分散资料、代码和运行线索，转化为可验证、可度量、可执行的项目认知与接手计划。

它解决的不是“把仓库内容重新排版”，而是帮助项目负责人回答六个关键问题：

1. 产品为谁解决什么问题，价值是否得到证据支持。
2. 当前实际上线了什么，哪些内容只是规划、推断或未知。
3. 模型、提示词、RAG、智能体、工具、数据和人工控制如何共同完成用户任务。
4. 质量、安全、成本、延迟、可靠性、采用和业务结果如何度量。
5. 哪些风险、依赖、产品债和知识缺口会影响后续决策。
6. 接手后的 30、60、90 天应该优先完成什么。

## 核心方法

项目采用 E0–E3 四级证据模型：

| 等级 | 含义 | 处理方式 |
|---|---|---|
| E3 · 已验证 | 多源证据一致，或被实现、运行数据直接验证 | 可作为当前事实使用 |
| E2 · 有依据 | 有一个当前强证据 | 保留来源并持续复核 |
| E1 · 推断 | 间接证据支持，但尚未确认 | 转化为验证任务或访谈问题 |
| E0 · 未知 | 缺少证据或证据互相矛盾 | 显式记录，不用假设补全 |

未知内容不会被隐藏，而是进入利益相关方议程、实验、埋点、评估、风险或路线图任务。

## 工作流程

```text
接手范围与目标
      ↓
资料盘点与证据台账
      ↓
产品 / 用户 / 旅程 / 能力重构
      ↓
AI 系统 / 数据 / 评估 / 安全审计
      ↓
指标 / 经济性 / 架构 / 运维分析
      ↓
风险 / 未知问题 / 产品债 / 协作责任
      ↓
30 / 60 / 90 天接手计划
      ↓
离线项目档案与自动验证报告
```

## 系统架构

项目分为四层：

1. **Skill 编排层：** 负责证据判断、产品分析、AI 系统审计、优先级和交付要求。
2. **证据模型层：** 维护资料清单、证据台账、置信度、矛盾、未知问题与负责人。
3. **确定性工具层：** 使用 Python 标准库完成脚手架、站点构建、链接重写和安全验证。
4. **离线交付层：** 提供 20 章可搜索、响应式、支持明暗主题和打印的项目接手档案。

这种分层让需要判断的工作由 AI 完成，把容易重复出错的构建和校验交给确定性脚本。

## 核心交付物

### 接手过程资料

`takeover-notes/` 保存接手简报、资料清单、证据台账、漂移登记表和未知问题。

### 项目接手档案

`ai-pm-dossier/` 覆盖产品、用户、AI 系统、数据与知识、评估与护栏、指标、经济性、架构、运维、路线图、产品债、风险、利益相关方议程和 30/60/90 天计划。

## 技术实现

- Skill 主体：`skill/ai-pm-project-takeover/SKILL.md`
- 方法参考：`skill/ai-pm-project-takeover/references/`
- 脚手架：`scaffold_dossier.py`
- 静态站点构建：`build_dossier.py`
- 结构、安全与离线验证：`verify_dossier.py`
- 完整示例：`examples/ai-pm-dossier-demo/`
- 自动化测试：`tests/test_dossier.py`
- 持续集成：`.github/workflows/ci.yml`

脚本只依赖 Python 标准库，生成站点不依赖服务端、数据库或远程 CDN。

## 质量保证

验证器检查 20 个必需章节、配置完整性、内容来源、死链、危险链接、远程资源、资源越界、常见秘密形态和生成结果。测试覆盖完整构建、非覆盖脚手架、章节缺失、未知页面、秘密检测、危险链接、远程样式和资源路径穿越。

示例站点同时完成桌面端、窄屏、目录交互、全文搜索和浏览器控制台检查。

## 使用方式

安装 Skill 后，在目标项目根目录调用：

```text
使用 $ai-pm-project-takeover 接手当前 AI 项目。
从产品、用户、业务、AI 系统、数据、评估、指标、架构、运维和组织协作角度完成系统尽调。
所有重要结论使用 E3/E2/E1/E0 标记，不要把推断写成事实。
生成 takeover-notes/、ai-pm-dossier/ 和 30 / 60 / 90 天接手计划。
```

完整参数和使用场景见 [使用说明](USAGE.md)。

## 设计边界

- 代码可以证明系统行为，不能单独证明用户价值。
- 测试通过不等于 AI 输出质量达标。
- 自动秘密检测不替代专业安全审查。
- 无法访问的用户研究、生产指标和组织知识必须标为未知。
- 目标项目默认只读，不自动执行部署、迁移、删除或生产写操作。
- 任何对外共享材料都必须完成独立脱敏和安全检查。
