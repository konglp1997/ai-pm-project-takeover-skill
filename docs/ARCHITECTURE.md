# 架构说明

## 分层

1. Skill 负责证据审计、内容组织和交付判断。
2. Python 脚本负责可确定、可重复的文件操作。
3. 静态资产负责浏览器端导航、搜索、主题和响应式布局。

构建工具没有第三方 Python 依赖，降低跨工作区运行成本。

## 组件

### `SKILL.md` 与 `references/`

`SKILL.md` 只保留路线和核心流程；资料审计、信息架构、编写规范和质量验收按需加载，避免每次触发占用过多上下文。

### `scaffold_handbook.py`

从 Skill 内置资产创建最小手册。所有写入都是“文件不存在才创建”，不会覆盖用户内容。

### `build_handbook.py`

```text
handbook.json + content/*.html + assets/
                    │
                    ├─ 校验配置与页面顺序
                    ├─ 重写 page:/asset: 链接
                    ├─ 注入标题锚点与页内目录
                    ├─ 生成侧边栏和上下页
                    ├─ 提取本地搜索索引
                    └─ 写入 index.html 与 pages/*.html
```

`handbook.json` 是导航和元数据的单一事实来源，`content/` 是正文来源，生成目录可安全重建。

### `verify_handbook.py`

同时检查源片段和最终站点：源检查给作者准确定位，输出检查验证相对路径和最终死链。秘密检测覆盖配置、源和生成页面，但属于启发式提醒，不是安全证明。

## 信任边界

正文片段被视为本地维护者输入。构建器不会清洗后改变语义；验证器禁止脚本、iframe、对象嵌入、事件处理器和远程资源，但不是完整 HTML 沙箱。

- 不要把不可信第三方 HTML 放入 `content/`。
- 不要把站点作为允许任意投稿的 CMS。
- 公开前人工检查正文和搜索索引。

## 链接与搜索

- `page:<slug>` 指向配置页面。
- `asset:<relative-path>` 指向站点 `assets/`。
- 构建器根据当前输出位置计算相对路径。
- 搜索索引从正文提取文本；标题、章节和摘要权重更高。

## 兼容性

- Python 3.9+。
- 现代 Chromium、Safari 或 Firefox。
- 不要求 HTTP 服务器，以 `file://` 为目标。
- Mermaid 是可选第三方运行时，不随仓库分发。
